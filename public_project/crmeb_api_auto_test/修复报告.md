## 1. headers 空值传入 requests 导致崩溃

**文件**：`tests/test_product.py`，第 38~50 行

**原代码**：
```python
header = case[ExcelVariable.case_headers]
if header:
    header = json.loads(case[ExcelVariable.case_headers])
    header["Authori-zation"] = get_token
    ...

response = ApiRequest.send_request(method, url, headers=header, ...)
```

**问题**：无鉴权用例的 headers 字段为空字符串 `''`，`if '':` 为假，header 保持为 `''`，最终传入 requests 的 `headers=''`。requests 要求 headers 为 dict 或 None，传空字符串会抛 `AttributeError`。项目 11 条用例中 7 条无鉴权，全部崩溃。

**修复后**（内联，在原第 38 行附近）：
```python
# 文件：tests/test_product.py，第 39~49 行
header = case[ExcelVariable.case_headers]
if header:
    try:
        header = json.loads(header)              # 内联 try/except
    except (json.JSONDecodeError, TypeError) as e:
        log.error(f'headers JSON解析失败: {e}, 原始值: {header}')
        header = None
    else:
        header["Authori-zation"] = f"Bearer {get_token}"
else:
    header = None          # ← 空值时显式设 None，不给 requests 传空串
```

## 2. `json.loads()` 无异常保护（3处）

**文件**：`tests/test_product.py`，原第 24/38/46 行

**原代码**：
```python
address_param = json.loads(address_param)    # 原第24行 — 地址参数
header = json.loads(case[...])               # 原第38行 — 请求头
query_params = json.loads(case[...])          # 原第46行 — 查询参数
```

**问题**：Excel 中手工填写的 JSON 字符串，多一个逗号、少一个引号即 `JSONDecodeError`。三处裸调，没有 try/except。

**修复后**（三处各自内联 try/except，不封装）：

```python

address_param = case[ExcelVariable.case_address_param]
if address_param:
    try:
        address_param = json.loads(address_param)
    except (json.JSONDecodeError, TypeError) as e:
        log.error(f'address_param JSON解析失败: {e}, 原始值: {address_param}')
        address_param = None

# ② 请求头 — tests/test_product.py，第 40~45 行（见修复1）

# ③ 查询参数 — tests/test_product.py，第 55~60 行
query_params = case[ExcelVariable.case_params]
if query_params:
    try:
        query_params = json.loads(query_params)

        log.error(f'query_params JSON解析失败: {e}, 原始值: {query_params}')

else:
    query_params = None
```

## 3. Token 注入丢失 Bearer 前缀

**文件**：`tests/test_product.py`，原第 39 行 → 修复后第 47 行

**原代码**：
```python
header["Authori-zation"] = get_token
```

**问题**：`get_token` fixture 返回裸 token，直接赋值。接口期望 `Bearer ` 前缀（YAML 中写的是 `"Authori-zation":"Bearer {token}"`）。丢失前缀后返回 401，5 条鉴权用例全部受影响。

**修复后**：
```python
# tests/test_product.py，第 47 行
header["Authori-zation"] = f"Bearer {get_token}"
```

## 4. jsonpath 结果直接 `[0]` 取下标

**文件**：`tests/test_product.py`，原第 53 行 → 修复后第 70~76 行

**原代码**：
```python
assert jsonpath(act_value, '$..message')[0] == case[ExcelVariable.case_expected]
```

**问题**：jsonpath 未匹配时返回 `False` 或空列表，`[0]` 直接 `IndexError`。

**修复后**（内联，在原位附近直接加 if/else）：
```python
# tests/test_product.py，第 70~76 行
act_message = jsonpath(act_value, '$..message')
if act_message:                     # 内联判断：列表非空
    act_message = act_message[0]    # 安全取 [0]
else:
    act_message = ''                # 未匹配时给空字符串
log.info(f'当前用例实际结果为{act_message}')
assert act_message == case[ExcelVariable.case_expected]
```

## 5. `r.json()` 无异常保护

**文件**：`tests/test_product.py`，原第 54 行 → 修复后第 65~69 行

**原代码**：
```python
act_value = r.json()
```

**问题**：接口返回 500 时 body 可能是 HTML 或纯文本，`r.json()` 抛 `JSONDecodeError`，掩盖真正的业务错误。

**修复后**（内联 try/except）：
```python
# tests/test_product.py，第 65~69 行
try:
    act_value = r.json()
except Exception as e:
    log.error(f'响应JSON解析失败: {e}')
    pytest.fail(f'响应体不是有效JSON: {r.text[:200]}')
```

## 7. conftest.py — 删除无效夹具 + Token 异常保护

**文件**：`conftest.py`

**原问题一**：`@pytest.mark.fixture` 装饰器不存在（应为 `@pytest.fixture`），`get_ids` 夹具调用 `/api/admin/product/list` 不带 token（必 401），且未被任何用例使用。

**原问题二**：`get_token` fixture 对登录失败无容错——网络不通、服务宕机、凭证过期时直接 KeyError/ConnectionError 崩溃，无日志。

**修复后**：
```python
# conftest.py，第 14~30 行
@pytest.fixture(scope='module', autouse=True)
def get_token():
    url = f'{BASE_URL}/api/front/login'
    json_data = {
        'account': LOGIN_ACCOUNT,
        'password': LOGIN_PASSWORD
    }
    try:
        response = requests.request(method='post', url=url, json=json_data, timeout=10)
        token = response.json()['data']['token']
        if not token:
            pytest.fail('登录成功但token为空')
        return token
    except requests.exceptions.RequestException as e:
        pytest.fail(f'登录请求失败: {e}')
    except (KeyError, json.JSONDecodeError) as e:
        pytest.fail(f'登录响应解析失败: {e}')
```

**变更清单**：
- 删除无效 `get_ids` 夹具
- `get_token` 增加 `try/except` 三级保护（RequestException / KeyError / JSONDecodeError）
- 增加请求超时设置（`timeout=10`）
- 增加 token 空值检查
- 补充 `import json`

## 8. `base/method.py` — 移除无用 `__init__`

**文件**：`base/method.py`

**原代码**：
```python
class ApiRequest:
    def __init__(self):
        self.responses = None       # ← 从未被使用
    @staticmethod
    def send_request(...):
        responses = requests.request(...)
        return responses
```

**问题**：`send_request` 是静态方法，`self.responses` 从未被使用。`__init__` 纯冗余。

**修复后**（移除 `__init__`）：
```python
# base/method.py
import requests
class ApiRequest:
    @staticmethod
    def send_request(method, url, params=None, data=None, headers=None, cookies=None,
                     files=None, auth=None, timeout=None, allow_redirects=True,
                     proxies=None, hooks=None, stream=None, verify=None, cert=None,
                     json=None):
        responses = requests.request(method, url, headers=headers, cookies=cookies,
                                     files=files, auth=auth, timeout=timeout,
                                     allow_redirects=allow_redirects, proxies=proxies,
                                     hooks=hooks, stream=stream, verify=verify, cert=cert,
                                     params=params, data=data, json=json)
        return responses
```
