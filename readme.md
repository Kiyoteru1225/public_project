# 接口自动化测试框架 
## 一、 技术栈 
‘pytest + requests + allure + ini + log 实现数据驱动框架，使用虚拟环境’ 
## 二、 搭建项目框架（创建**_api_auto_test_**目录） 
依次在项目下创建目录  
test：存放测试用例，称之为用例层  
data：存放测试数据，称之为数据层，测试用例用excel/yaml管理。excel一个sheet页对应一个接口模块，一个yaml文件对应一个接口模块  
tools：存放一些公共方法，例如：获取文件绝对路径、解析数据格式等  
base：存放一些最底层的方法封装、协议、请求方法等  
report：存放测试报告  
logs：存放运行日志  
config：存放环境mysql的配置信息  
main.py：程序主入口  
conftest.py：存放公用测试夹具  
pytest.ini：用于编写pytest参数和配置文件  
requirements.txt：项目依赖包版本信息，依赖三方库  
readme.md：项目说明文档  
## 三、 使用说明 
### 第一步
首先需要通过pip命令安装依赖包  
运行main.py文件即可  
增加用例可写入excel/yaml文件  
用例需要特殊规则需在用例层的模块中特殊化处理  
### 第二步 封装请求方法
在base目录下创建method.py模块
直接调用requests库里面的request方法，并将所需要的参数定义进去，分为实参和形参，实参调用必须要传递参数，形参可以给默认值，调用时可重新赋值，也可以使用默认值，request会根据传入参数发送请求到服务器
### 第三步 封装读取文件的方法
在tools目录下创建两个文件  
- path_utils.py模块封装获取文件绝对路径的方法。直接调用，传入相应文件名参数可直接使用  
- data_loader.py封装读取、解析文件数据格式的方法，直接返回解析后的数据。获取单表或yaml文件的全部数据，按约定格式处理。  
### 第四步 编写配置文件
- 配置接口服务信息，并解析配置文件数据  
- 在config目录下新建配置文件.ini并编写接口服务器相关信息，配置文件包含两个部分：章节和变量  
- 在tools数据解析模块中定义解析ini文件方法  
### 第五步 对excel表头定义全局变量
- 在tools数据解析模块中定义解析excel表头方法  
### 第六步 封装登录方法获取 token 信息
- 大部分接口公共依赖token认证，封装登录方法获取token信息  
- 代码写入到config.py模块中  
- 使用pytest提供的装饰器@pytest.fixture，每次执行前获取一次token信息  
### 第七步 创建主程序入口
- 创建main.py模块，作为项目主入口，项目的根目录下创建  
- 主要收集测试用例，并执行  
- 生成html测试报告  
### 第八步 修改pytest运行规则
- 创建pytest.ini文件，并编写运行参数和配置文件  
### 第九步 封装用例
- 封装用例在tests目录下
- 需要调用封装的数据解析方法，获取数据
- 需要调用封装的请求方法，发送请求
- 获取的数据用例存放进参数化装饰器中，data为别名，用例方法中对接口地址、请求头、参数等进行特殊化处理
- 将token赋值给header，不用每次都去调用夹具获取token
- 一个功能模块对应一个用例文件，及一个模块对应一个excel/yaml文件
