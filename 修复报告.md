## 1. headers 空值传入 requests 导致崩溃

**原代码**：
```python
header = case[ExcelVariable.case_headers]
if header:
    header = json.loads(case[ExcelVariable.case_headers])
    header["Authori-zation"] = get_token
    ...

response = ApiRequest.send_request(method, url, headers=header, ...)
```

**问题**：无鉴权用例的 headers 字段为空字符串 `''`，`if '':` 为假，header 保持为 `''`，最终传入 requests 的 `headers=''`。requests 要求 headers 为 dict 或 None，传空字符串会抛 `AttributeError`。项目 11 条用例中 7 条无鉴权，全部崩溃。

**修复后**：
```python
headers = _safe_json_parse(case[ExcelVariable.case_headers], "请求头")
if headers and isinstance(headers, dict):
    headers["Authori-zation"] = f"Bearer {get_token}"
else:
    headers = None  # 无鉴权接口传None
```

## 2. `json.loads()` 无异常保护（3处）

**原代码**（第24/38/46行）：
```python
address_param = json.loads(address_param)    # 第24行
header = json.loads(case[...])               # 第38行
query_params = json.loads(case[...])          # 第46行
```

**问题**：Excel 中手工填写的 JSON 字符串，多一个逗号、少一个引号即 `JSONDecodeError`。三处裸调，没有 try/except。

**修复后**：新增 `_safe_json_parse()` 工具函数，统一处理 JSON 解析 + 空值过滤 + 异常捕获，解析失败返回 None 并记录警告。

```python
def _safe_json_parse(raw_value, field_name="未知字段"):
    if not raw_value or (isinstance(raw_value, str) and raw_value.strip() == ''):
        return None
    if isinstance(raw_value, dict):
        return raw_value
    try:
        return json.loads(raw_value)
    except (json.JSONDecodeError, TypeError) as e:
        log.warning(f"[{field_name}] JSON解析失败: {e}")
        return None
```

## 3. Token 注入丢失 Bearer 前缀

**原代码**：
```python
header["Authori-zation"] = get_token
```

**问题**：`get_token` fixture 返回裸 token，直接赋值覆盖了整个值。YAML 中写的是 `"Authori-zation":"Bearer {token}"`，说明接口期望 `Bearer ` 前缀。丢失前缀后，符合 OAuth2 规范的服务端会返回 401。5 条鉴权用例全部受影响。

**修复后**：
```python
headers["Authori-zation"] = f"Bearer {get_token}"
```

同时兼容 YAML 中已写 `Bearer {token}` 占位符的场景：如果 header 中已有 `{token}` 占位符则替换，否则追加完整 `Bearer xxx`。

## 4. jsonpath 结果直接 `[0]` 取下标

**原代码**：
```python
assert jsonpath(act_value, '$..message')[0] == case[ExcelVariable.case_expected]
```

**问题**：jsonpath 未匹配时返回 `False` 或空列表，`[0]` 直接 `IndexError`。接口变更字段名（如 `msg` 替代 `message`）时全量崩溃。

**修复后**：新增 `_safe_jsonpath_extract()` 函数统一处理，先检查结果是否有效，再断言非空。

```python
def _safe_jsonpath_extract(data, expr):
    try:
        result = jsonpath(data, expr)
        if result is False or result is None:
            return []
        return result
    except Exception:
        return []

messages = _safe_jsonpath_extract(act_value, '$..message')
assert len(messages) > 0, f"jsonpath '$..message' 未匹配到任何值"
assert messages[0] == expected
```

## 5. `r.json()` 无异常保护

**原代码**：
```python
act_value = r.json()
```

**问题**：接口返回 500 时 body 可能是 HTML 或纯文本，`r.json()` 抛 `JSONDecodeError`，掩盖真正的业务错误。

**修复后**：
```python
try:
    act_value = r.json()
except (json.JSONDecodeError, ValueError, requests.exceptions.JSONDecodeError) as e:
    log.error(f"响应体非JSON格式: {e}")
    pytest.fail(f"响应体非JSON: {r.text[:200]}")
```

## 6. jsonpath 结果被丢弃未赋值（低）

**原代码**（第53-54行）：
```python
jsonpath(act_value, '$..message')                              # 白算
log.info(f'当前用例实际结果为{jsonpath(act_value, "...")}')      # 又算一遍
```

**问题**：第53行调用 jsonpath 但未赋值，白白浪费一次遍历。第54行 log 中又算了一遍。虽然不影响功能，但属于编码疏忽。

**修复后**：由 `_safe_jsonpath_extract()` 统一提取并复用。

## 7. conftest.py — 删除无效夹具 + Token 异常保护

**原问题一**：`@pytest.mark.fixture` 装饰器不存在（应为 `@pytest.fixture`），`get_ids` 夹具调用 `/api/admin/product/list` 不带 token（必 401），且未被任何用例使用。

**原问题二**：`get_token` fixture 对登录失败无容错——网络不通、服务宕机、凭证过期时直接 KeyError/ConnectionError 崩溃，无日志。

**修复**：
- 删除 `get_ids` 夹具
- `get_token` 增加 `try/except`，捕获请求异常和解析异常，记录日志并用 `pytest.fail()` 明确终止
- 增加 token 为空检查
- 增加请求超时设置（timeout=10）

## 8. base/method.py — 移除无用 `__init__`

**原代码**：
```python
class ApiRequest:
    def __init__(self):
        self.responses = None
    @staticmethod
    def send_request(...):
        responses = requests.request(...)
        return responses
```

**问题**：`send_request` 是静态方法，`self.responses` 从未被使用。`__init__` 纯冗余。

**修复后**：移除 `__init__`，保留纯静态方法。添加方法文档字符串。
